# CookCook 开发计划书

## 总体要求
1. **缓存刷新机制**：每次更新 CSS 或 JS 文件时，通过添加版本号参数（如 `?v=1.0.1`）强制刷新浏览器缓存。
2. **多端代码同步**：每次修改后，代码需自动 commit 并推送到 Gitee 和 GitHub 两个仓库。将配置快捷脚本来实现一键双推。
3. **分阶段发布策略**：
   - 第一阶段（当前）：先不接入 OSS。在 `_uploader` 文件夹中使用 Python 脚本将 `asset` 里的多媒体资源与 JSON 格式的菜谱信息进行解析和上传，生成初始测试数据。
   - 第二阶段：添加前端“发布菜单”功能，数据仅保存在浏览器缓存（刷新失效），以验证数据流转和交互体验。
   - 第三阶段：正式接入 OSS（对象存储服务），增加二级密码验证机制，实现真实完整的发布功能与云端存储。

## 阶段一详细任务规划（进行中）

### 1. 基础工程搭建与资源准备
- **目录结构建立**：规划基础的 HTML/CSS/JS 结构。
- **头像资源迁移**：从 `D:\seikai_cn\dandawang\public\avatars` 复制用户头像到本项目。
- **GitHub Pages 配置**：建立触发 GitHub Pages 部署的文件配置。
- **双端推送脚本**：编写 `push.ps1` 脚本，完成向 Gitee 和 GitHub 的一键 commit & push。

### 2. 测试数据预处理系统 (`_uploader` 架构)
- 创建 `_uploader/asset` 文件夹用于存放封面图和视频教程。
- 编写 `_uploader/recipes.json`：填写包含所有必要信息（包含各种步骤类型等）的测试菜谱数据。
- 编写 `_uploader/upload.py`：实现 Python 脚本，模拟解析并转换为前端可用的菜谱数据格式 `data.js`。

### 3. 前端界面与功能开发（前两个核心界面）
- **首页登录界面**：
  - 高级视觉设计，动态动画效果，吸引眼球（深色模式、毛玻璃效果等）。
  - 登录表单交互，模拟登录状态管理。
- **主页与选菜界面**：
  - **搜索与排序**：支持关键字搜索菜谱，按时间、热度等维度排序。
  - **选菜浏览**：展示菜谱列表，包含封面、标题、简短介绍等信息。
  - **所有功能实现**：遵循组件化和模块化的原则实现，CSS/JS 加版本号控制。
